# Backrooms Architecture Design Guide

> **프로젝트 최상위 설계 기준 문서**  
> Level 0(클래식 백룸) 레퍼런스 이미지 8장을 건축학적 관점에서 교차 분석한 결과입니다.  
> 본 문서는 **던전 생성 규칙이 아니라**, 오래된 사무실·상업 층을 절차적으로 재현하기 위한 건축 설계 기준입니다.

**분석 대상:** 제공된 레퍼런스 8장 (사용자 언급 10장 중 8장 분석 가능)  
**분석 방법:** 이미지별 특징 추출 → 2장 이상 공통 출현 여부 확인 → 1장 한정 특징 제외  
**코드 상태:** 본 문서 작성 시점에 **게임 코드는 수정하지 않음**

---

## 0. Executive Summary

백룸 Level 0의 핵심은 **「좁은 복도 미로」가 아니라 「넓은 사무 층을 벽 파티션이 느슨하게 나눈 공간」**이다.

| 항목 | 백룸 (공통) | 전형적 던전 (지양) |
|------|-------------|-------------------|
| 중심 공간 | 라운지·오픈 플로어 | 복도 + 작은 방 |
| 연결 | 개방형 개구부, 문 없음 | 문·게이트·좁은 통로 |
| 벽 역할 | 파티션·시선 차단 | 외벽·미로 벽 |
| 시야 | 여러 구역이 동시에 보임 | 한 방향 시야 제한 |
| 분위기 | 익숙하지만 비어 있음 | 인위적·게임적 |

## 1. 방과 복도의 비율

### 1.1 공통 결론

- **복도는 소수.** 8장 중 **긴 복도가 주제인 이미지는 1장**(약 12%)뿐이다. 나머지 7장은 **넓은 오픈 플로어·라운지형** 공간이 지배적이다.
- **「방」과 「복도」의 이분법이 성립하지 않는다.** 대부분의 이미지에서 공간은 **방/복도가 아니라 「구역(zone)」**으로 인식된다. 벽 배치가 시야와 동선만 부분적으로 가른다.
- **플레이어 체감 비율 (추정):**
  - **넓은 개방 구역:** 60–75%
  - **중간 폭 통로** (벽 사이 3–6m급): 20–30%
  - **긴 1차원 복도** (한 축으로 길게 늘어진 통로): **≤15%**, 이미지마다 주역이 아님

### 1.2 공간이 이어지는 방식

```
[넓은 라운지] ──개구부──▶ [중간 구역] ──개구부──▶ [또 다른 넓은 구역]
        │                         │
        └── 파티션 벽(부분) ──────┘
```

- 공간은 **그래프(tree/mesh)가 아니라 연속 바닥 위의 파티션 배치**처럼 느껴진다.
- 한 지점에서 **2개 이상의 넓은 영역이 동시에 보이는 경우**가 반복적으로 등장한다 (시선 연속성).
- 「방 → 방 → 넓은 공간 → 방 → 방」 리듬이 레퍼런스 전반의 동선 패턴과 일치한다.

### 1.3 복도에 대한 정량적 기준 (생성 목표)

| 지표 | 권장 범위 | 근거 |
|------|-----------|------|
| 복도 셀(통로) 비율 | **≤15–20%** | 8장 중 7장이 복도 중심이 아님 |
| 최소 통로 폭 | **≥3m** (이상적으로 플레이어 몸통 2배 이상) | 좁은 던전 통로(1–1.5m)는 레퍼런스에 거의 없음 |
| 연속 복도 최대 길이 | **짧게 유지**, 라운지로 자주 개방 | 긴 1점 투시 복도는 예외적 연출 |

## 2. 방의 형태

### 2.1 가장 많이 등장하는 형태

**공통 순위 (출현 빈도):**

1. **직사각형 오픈 존** — 가장 빈번
2. **L자형 구역** — 벽 한두 면이 꺾이며 자연 발생
3. **T자 교차 구역** — 파티션 끝이 다른 벽과 만나며 발생
4. **ㄷ자·ㄹ자** — 명시적 ㄷ자는 드물지만, **벽 배치로 우회 동선**이 생기는 패턴은 다수

**한 장에서만 보이는 형태 (생성 필수 아님):**

- 허리 높이 **반파티션(half-wall)** — 1장
- 리벳이 박힌 **구조 기둥** — 1장
- 의자 더미 등 가구 배치 — 연출 요소, 건축 골격 아님

### 2.2 방 크기 분포

레퍼런스 전반에서 관측되는 **상대 스케일** (천장고 ~2.7–3.0m 기준):

| 유형 | 상대 크기 | 비고 |
|------|-----------|------|
| 소형 알코브 | 3m × 3m ~ 4m × 5m | 벽 파티션으로 생긴 작은 움푹한 공간 |
| 중형 구역 | 5m × 6m ~ 8m × 10m | 가장 흔한 체류 공간 |
| 대형 라운지 | 10m × 12m 이상 | 시야가 열리고 조명이 넓게 퍼짐 |
| 긴 축 공간 | 한 축 12m+, 다른 축 4–6m | 「긴 방」에 가깝고 복도와 다름 (폭이 넓음) |

**크기 분포 목표:** 소:중:대 ≈ **2:5:3** (중·대형 비중이 높음)

### 2.3 긴 방 vs 넓은 방

- **넓은 방(양축 모두 큼)**이 **긴 복도형 공간**보다 훨씬 많다.
- 「긴 방」은 존재하지만 **폭이 좁지 않다** (최소 4m 이상).
- **가로:세로 비율** 대부분 **1:1 ~ 1:2** — 1:4 이상의 띠형 공간은 복도 예외 이미지 외에는 드묾.

### 2.4 직교 구조의 특징

- **100% 직교(90°)** — 대각선 벽, 곡선, 경사 없음
- 벽은 **완전한 폐실 방**보다 **1~3면이 열린 파티션**인 경우가 많음
- 코너는 **내부 코너(concave)** 와 **외부 코너(convex)** 가 혼재하나, **모두 날카로운 직각**
- 격자 정렬: 벽, 천장 타일, 조명이 **동일 축에 정렬** (8/8)

## 3. 공간 연결 방식

### 3.1 방과 방의 연결

**공통 패턴 (7/8장 이상):**

- **문(door)이 없다** — 문틀, 손잡이, 개폐식 구조물 없음
- 연결은 **벽이 끝나며 생기는 개방형 개구부(opening)** 로 이루어진다
- 개구부는 **프레임 없는 직사각형 절개**처럼 보인다 (벽 두께 면만 노출)

**예외적 표현 (1장):**

- 「쪽문(doorway)」처럼 보이는 개구부 — 그러나 **문짝·문틀 없이** 벽 두께만 보이는 형태

### 3.2 개방 vs 폐쇄

| 연결 유형 | 빈도 | 설명 |
|-----------|------|------|
| 완전 개방 (벽 없음) | 높음 | 넓은 라운지가 한 바닥처럼 이어짐 |
| 부분 파티션 | 높음 | 벽이 시야·동선만 일부 차단 |
| 개구부 통과 | 중간 | 벽에 큰 구멍, 양쪽 공간이 보임 |
| 좁은 문형 통로 | **낮음** | 전형적 사무실 문보다 훨씬 넓음 |

### 3.3 긴 복도의 실제 등장

- **주 공간으로서의 긴 복도: 드묾** (8장 중 1장)
- 대부분 이미지에서 「복도」는 **넓은 층 내 통로** 수준이며, **전용 미로 복도가 아님**
- 생성기는 **복도를 생성 목표로 삼지 말고**, 라운지 사이 **자연스럽게 남는 통로**만 허용

### 3.4 연결 규칙 요약

```
✓ 벽 일부 제거(subtractive opening)로 연결
✓ 개구부 폭 ≥ 2m (넓을수록 백룸스러움)
✓ 연결 지점에서 양쪽 바닥·천장 높이 동일
✗ 문짝·문틀·키홀
✗ 1m급 좁은 게이트
✗ 일방통행 느낌의 시퀀스 방
```

## 4. 건축 특징

### 4.1 벽 두께

**공통 관측 (다수 이미지 합의):**

- **실제 사무실 파티션 두께: 약 15–20cm (0.15–0.20m)**
- 일부 이미지에서 벽이 더 두꺼워 보이는(0.5–1m) 경우는 **시각적 덩어리감·파티션 블록** 연출이며, 모든 장에서 동일하지 않음
- **생성 기준 두께: 16cm 전후** (현재 프로젝트 `WALL_T = 0.16`과 일치) — 물리적으로 타당

**규칙:**

- 두께는 **전 구간 일정** — 코너에서만 두꺼워지거나 얇아지면 안 됨
- 개구부 가장자리에서 **벽 단면(end cap)이 명확히 보임** — 얇은 면 없음

### 4.2 코너 처리

**8/8 공통:**

- **90° 날카로운 코너** — 모따기·모딩 라운드 없음
- **코너 틈(gap) 없음** — 벽이 건축적으로 맞물림
- **기둥처럼 튀어나온 코너 없음** (올바른 레퍼런스) — 두께가 겹쳐 부풀어 오르지 않음
- 내부 코너·외부 코너 모두 **동일 재질·동일 패턴**으로 연속

### 4.3 천장 구조

**8/8 공통:**

- **매립형 천장(drop ceiling / suspended ceiling)**
- **사각 격자 타일** — 어쿠스틱 패널
- 격자선이 **벽 축과 평행** 정렬
- 천장고 **약 2.7–3.0m** (표준 오피스)

### 4.4 조명 배치

**8/8 공통:**

- **형광 트로퍼(troffer)** — 천장 격자에 매립된 직사각 패널
- 배치는 **완벽한 단일 격자보다 약간 불규칙**한 경우도 있으나, **행(row) 패턴**은 유지
- **따뜻한 황록·담황색** 인공광 — 자연광 없음
- 그림자는 **부드럽고 확산** — 극단적 대비 없음

### 4.5 몰딩 유무

**이미지 간 불일치 → 공통 규칙으로 정리:**

| 유형 | 출현 | 결론 |
|------|------|------|
| 걸레받이(baseboard) | 2장 유 / 5장 무 / 1장 불명확 | **필수 아님** — Level 0 클래식은 **몰딩 없는 날카로운 접합**이 더 지배적 |
| 천장 몰딩(crown) | 대부분 없음 | **생성하지 않음** |
| 반파티션 상단 캡 | 1장 | 선택적 변형, 기본 규칙 아님 |

**생성 기준:** 기본은 **바닥·천장 몰딩 없음**, 벽-바닥·벽-천장 **직각 맞댐**

### 4.6 현실 건축물과의 공통점

- 1970–1990년대 미국·유럽 **상업 오피스 층** 전형
- 개방형 사무실 + **가벼운 건식 경량벽(drywall partition)**
- **이동식 칸막이 느낌**의 파티션 배치 (완전한 실별화 아님)
- **창문 없음** (8/8) — 내부 코어·지하 또는 비현실적 층
- **가구 없음 또는 산발적** — 사용 중단된 사무공간
- 바닥: **통일된 저이 카펫** — 이음새 최소

## 5. 벽지 분석

### 5.1 패턴 방향

**8/8 공통:**

- 패턴의 **주축은 수직(vertical)**
- 미세한 **세로 줄무늬·화살표(chevron)·다마스크** 등 저대비 반복
- 색상: **담황·베이지·황록** 단색조 — 고대비 프린트 없음

### 5.2 UV 연속 방식

**8/8 공통 (가장 중요한 시각 규칙):**

- 벽지는 **개별 박스·개별 면 단위 타일링이 아님**
- **월드 좌표 기준(world-space UV)** 또는 이에 준하는 연속 투영
- **모든 벽 면에서 세로 패턴이 중력 방향(월드 Y)과 정렬**
- 천장고에 따라 **V(높이) 방향 스케일 일정** — 늘어남·찌그러짐 없음

### 5.3 코너에서의 패턴 연결

**8/8 공통:**

- **내부 코너:** 패턴이 꺾이며 **끊김·회전·스케일 변화 없이** 이어짐
- **외부 코너:** 바깥 모서리를 **감싸며(wrap)** 동일 패턴 유지
- 코너에서 **별도 텍스처·회색 캡·90° 회전 타일** 없음

### 5.4 벽 두께(단면) 재질

**8/8 공통:**

- 벽 **앞면·뒷면·단면(end cap)·코너 면** 모두 **동일 벽지**
- 단면에서 패턴이 **앞면과 정렬** — 다른 재질·단색 캡 없음
- 개구부 **잼브(jamb)** 에도 동일 벽지

### 5.5 벽지 금지 사항

```
✗ 코너마다 UV 90° 강제 회전
✗ 벽 두께 면에 회색/플레인 캡
✗ 면마다 다른 repeat 스케일
✗ 랜덤 오프셋으로 패턴 어긋남
✗ 조명만 다른 것처럼 보이는 텍스처 단절
```

| T5 | 천장 드롭 타일 + 형광 트로퍼 | 8/8 |
| T6 | 창문 없음 | 8/8 |

### 6.5 제외된 특징 (1장 한정 또는 비핵심)

- 허리 높이 반파티션 + 상단 캡
- 리벳 구조 기둥
- 걸레받이 **있는** 버전 (일부 장)
- 의자 더미·콘센트 등 소품
- 극단적으로 두꺼운(50cm+) 블록 벽 연출

## 7. 생성 규칙 (Procedural Office Generator Rules)

아래 규칙은 앞으로 **랜덤 맵 생성기가 반드시 따라야 하는 건축 규칙**이다.

### 7.1 공간 비율 규칙 (Space Budget)

```
ROOM_FLOOR_TARGET     ≥ 80%   # 바닥 면적 중 체류 가능 존
CORRIDOR_LIKE_MAX     ≤ 18%   # 좁은 통로형 셀
LOUNGE_MIN_RATIO      ≥ 25%   # 최소 1개 대형 연결 구역
MIN_OPENING_WIDTH     ≥ 2.0m  # 연결 개구부
PREFERRED_OPENING     ≥ 2.5m  # 플레이어 몸통 여유
```

- **복도 우선 생성 금지** — MST 복도 캐러버 금지, 라운지 팩 후 **부족 연결만 개구부로**
- 동선 그래프는 **단일 연결 컴포넌트** — 고립 방 없음

### 7.2 존(Zone) 배치 규칙

1. **시드 라운지** — 청크 중앙 또는 무게중심에 **가장 큰 존** 배치
2. **인접 부착** — 새 존은 기존 바닥에 **변(edge)을 공유**하며 붙음 (겹침 없음)
3. **형태 템플릿 가중치:**
   - 직사각 35%
   - L자 25%
   - 넓은 라운지 20%
   - 긴 직사각(폭≥4m) 10%
   - ㄷ·ㄹ·T 변형 10%
4. **긴 띠형(폭 < 3m) 존 생성 금지** — 복도화 방지

### 7.3 연결 규칙

```
CONNECTION_MODE       = OPENING_ONLY    # 문 메시·문틀 없음
OPENING_IN_PARTITION  = true            # 공유 파티션 벽 제거
CHUNK_EDGE_OPENING    = true            # 청크 경계도 개구부 (문짝 X)
MIN_CONNECTIONS_PER_ZONE ≥ 1
```

- 존 간 연결: **공유 벽에 개구부** (최소 통과 폭 `MIN_OPENING_WIDTH`)
- 연결 개수: **MST + 30–45% 추가 루프** — 완전 미로가 아닌 **여러 경로 가능한 사무 층**
- 개구부는 **문 데이터가 아니라 벽 세그먼트 부재(gap)** 로 표현

### 7.4 벽 생성 규칙 (Architectural Wall System)

```
WALL_THICKNESS        = 0.16m   # 전 구간 일정
WALL_MESH_MODE        = CONTINUOUS_FOOTPRINT
CORNER_MODE           = IMPLICIT_UNION   # 박스 덧붙이기 금지
CONNECTOR_ANALYSIS    = true    # 정점 차수로 연결부 분류
```

**필수:**

- 벽 외곽선 기반 **단일 연속 메시**
- 코너: Straight / End Cap / Inner·Outer Corner / T / Cross / Opening — **기하학적 연속체의 일부**
- 생성 후 검증:
  - 돌출(protrusion) 셀 0
  - 코너 두께 편차 ≤ 1mm
  - dangling segment 0
  - 개구부 잼브 면 누락 0

**금지:**

- `BoxGeometry` 이어 붙이기
- 코너에 별도 기둥 박스
- 벽 연장·겹침으로 코너 메우기
- 내부 면(솔리드 내부) 렌더

### 7.5 벽지·UV 규칙

```
UV_MODE               = WORLD_SPACE
PATTERN_AXIS          = VERTICAL (world Y aligned stripe)
WALL_MATERIAL         = SINGLE_WALLPAPER
THICKNESS_FACES       = SAME_MATERIAL
JAMB_FACES            = SAME_MATERIAL
TILE_WIDTH            = 0.76m   # 사용자 제공 벽지 기준
```

- U축: 면 법선에 따라 `worldX` 또는 `worldZ` / `tileW`
- V축: `worldY` / `tileH`
- **청크 경계를 넘어 UV 연속** (`roomWx`, `roomWz` 오프셋)
- 코너에서 **정점 공유 + 동일 UV 투영** — 패턴 꺾임은 3D 기하에 따른 자연스러운 wrap

### 7.6 천장·바닥·조명 규칙

| 요소 | 규칙 |
|------|------|
| 바닥 | walkable 존만 카펫 (이상적) — 전체 청크 카펫은 레퍼런스와 부분 불일치, 개선 목표 |
| 천장 | 드롭 타일 격자, `PANEL_SIZE` 정렬 |
| 조명 | 트로퍼 인스턴스, 격자 스냅, 황색광 |
| 몰딩 | 기본 없음 |

### 7.7 검증 파이프라인 (생성 후 자동)

```
1. Grid connectivity flood  → 단일 바닥 컴포넌트
2. Passage width check      → 모든 개구부 ≥ MIN_OPENING_WIDTH
3. Room/corridor ratio      → corridor ≤ 18%
4. Wall footprint QA        → protrusion, gap, thickness
5. Connector classification → 모든 정점 타입 기록
6. Nav flood from chunk exits → 4방향 출구 연결
```

실패 시: **재생성(최대 N회) → repair pass → fallback 단일 라운지**

## 8. 현재 프로젝트와의 Gap (참고, 구현 아님)

본 섹션은 향후 구현 시 우선순위 참고용이며, **이 문서 단계에서는 코드 변경 없음**.

| 설계 기준 | 현재 프로젝트 추정 상태 |
|-----------|-------------------------|
| 라운지 중심 ≥80% | `roomSpaceOK` 80% — **부합** |
| 복도 ≤18% | `corridorBudgetOK` — **부합** |
| 개방형 연결 | `openEdges` 파티션 제거 — **부합** |
| 연속 벽 메시 | footprint union 도입 — **방향 부합**, 품질 검증 지속 필요 |
| 월드 UV | 구현됨 — **부합** |
| 문/문틀 없음 | 청크 edge door gap은 있으나 문짝 없음 — **부합** |
| 바닥 walkable만 | 전체 청크 카펫 — **Gap** |
| 몰딩 없음 | 몰딩 없음 — **부합** |
| 반파티션 | 미구현 — **선택 사항** |

## 9. 용어 정의

| 용어 | 정의 |
|------|------|
| **Zone(구역)** | 바닥 패킹의 하나의 연결된 직교 영역. 완전 폐실이 아님 |
| **Lounge(라운지)** | 면적이 큰 zone. 체류·시야의 허브 |
| **Opening(개구부)** | 파티션 벽이 없는 연결 — 문이 아님 |
| **Partition(파티션)** | 바닥부터 천장까지의 경량 내벽 |
| **Connector(연결부)** | 벽 정점에서의 기하학적 연결 유형 — 별도 오브젝트 아님 |
| **Corridor-like** | 폭 < 3m 이고 길이/폭 > 2 인 띠형 공간 |

- 벽 두께 오류 ❌
- 벽지 패턴 끊김 ❌
- UV 깨짐 ❌
